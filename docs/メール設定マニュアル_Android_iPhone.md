# スマートフォン メール設定マニュアル（Android / iPhone）

**株式会社ヴィレッジ　御中**
**対象メール：`@village2024.jp`（CoreServer）**

---

このマニュアルは、独自ドメイン（`village2024.jp`）のメールを
**スマートフォン（iPhone / Android）で送受信できるようにする**ための設定手順書です。

設定はおおむね **5〜10分** で完了します。
途中で「サーバー名」「メールアドレス」「パスワード」を入力する場面があるため、
先に次の **「① 設定情報シート」** を確認してから始めてください。

> **受信方式について**
> 本マニュアルは **IMAP（アイマップ）方式** を推奨設定としています。
> IMAP は、サーバー上でメールを管理する方式です。
> スマホ・パソコン・複数の端末で **同じ受信トレイを共有** でき、機種変更時もメールが消えません。
> 会社のメールはこの方式が安心です。

## ① 設定情報シート（最初に確認してください）

| 項目 | 入力する値 |
|------|-----------|
| **メールアドレス** | `info@village2024.jp` |
| **パスワード** | メールアカウントのパスワード（CoreServer 管理画面で確認・再設定可） |
| **受信サーバー名** | `v2008.coreserver.jp` |
| **送信サーバー名** | `v2008.coreserver.jp`（受信と同じ） |
| **ユーザー名** | `info@village2024.jp`（メールアドレスと同じ） |

> **メモ**
> サーバー名は受信・送信とも共通で `v2008.coreserver.jp` です。
> 独自ドメイン名（`village2024.jp`）ではなく、この `coreserver.jp` で終わるサーバー名を使ってください。

### サーバー設定の固定値（CoreServer 標準・変更不要）

| 区分 | 項目 | 値 |
|------|------|-----|
| **受信（IMAP）** | ポート番号 | **993** |
| | 暗号化（SSL/TLS） | **使用する（オン）** |
| **送信（SMTP）** | ポート番号 | **465** |
| | 暗号化（SSL/TLS） | **使用する（オン）** |
| | 送信時の認証 | **必要（オン）／受信と同じユーザー名・パスワード** |

> ※ポート 993／465 がうまくいかない場合の代替値：受信 IMAP は **143**、送信 SMTP は **587**
> （その場合、暗号化は「**STARTTLS**」を選んでください。※`143`＋`SSL/TLS` の組み合わせは接続できません）

## ② iPhone（iPad）の設定手順

iPhone 標準の「**メール**」アプリで受信できるようにします。

### 手順

1. ホーム画面の「**設定**」（歯車アイコン）を開きます。
2. 下にスクロールして「**アプリ**」→「**メール**」を選びます。
   - （iOS のバージョンによっては「設定」内に直接「**メール**」がある場合があります）
3. 「**メールアカウント**」→「**アカウントを追加**」をタップします。
4. アカウントの種類で「**その他**」を選びます。
5. 「**メールアカウントを追加**」をタップします。
6. 次の項目を入力し、右上の「**次へ**」をタップします。
   - **名前**：差出人として表示される名前（例：ヴィレッジ）
   - **メール**：`info@village2024.jp`
   - **パスワード**：メールのパスワード
   - **説明**：分かりやすい名前（例：ヴィレッジメール）
7. 画面上部で「**IMAP**」が選ばれていることを確認します。
8. 「**受信メールサーバ**」に次を入力します。
   - **ホスト名**：`v2008.coreserver.jp`
   - **ユーザ名**：`info@village2024.jp`（全部入力）
   - **パスワード**：メールのパスワード
9. 「**送信メールサーバ**」に次を入力します（受信と同じ内容）。
   - **ホスト名**：`v2008.coreserver.jp`（受信と同じ）
   - **ユーザ名**：`info@village2024.jp`（全部入力）
   - **パスワード**：メールのパスワード
   - ※「オプション」と書かれていても、ユーザ名・パスワードは**必ず入力**してください。
10. 右上の「**次へ**」をタップします。確認に少し時間がかかります。
11. 「**保存**」をタップして完了です。

> **うまくいかないとき（iPhone）**
> 「サーバの識別情報を検証できません」と出たら「**続ける**」を押して進めてください。
> それでも保存できない場合は、
> 「設定」→「メール」→「アカウント」→ 作成したアカウント →「**SMTP**」→ サーバー名をタップし、
> **使用するポート＝465／SSLを使用＝オン**、または代替の **587（STARTTLS）** になっているかを確認します。

## ③ Android の設定手順

Android は機種（メーカー）によって標準メールアプリが異なるため、
**どの機種にも入っている「Gmail」アプリ**を使う方法で説明します。
（普段 Gmail を使っていなくても、独自ドメインのメールを Gmail アプリで受信できます）

### 手順

1. 「**Gmail**」アプリを開きます。
2. 右上の「**アカウントのアイコン（丸い人型）**」をタップします。
3. 「**別のアカウントを追加**」をタップします。
4. メールのセットアップ画面で「**その他**」を選びます。
5. **メールアドレス**（`info@village2024.jp`）を入力し、「**次へ**」をタップします。
6. アカウントの種類で「**個人用（IMAP）**」を選びます。
7. **パスワード**を入力し、「**次へ**」をタップします。
8. 「**受信サーバーの設定**」を入力します。
   - **ユーザー名**：`info@village2024.jp`（全部入力）
   - **パスワード**：メールのパスワード
   - **サーバー**：`v2008.coreserver.jp`
   - 画面下に出るポート／セキュリティは **ポート993／SSL（TLS）** にします。
   - 入力できたら「**次へ**」。
9. 「**送信サーバーの設定**」を入力します。
   - **「ログインが必要」をオン**にします。
   - **ユーザー名**：`info@village2024.jp`（全部入力）
   - **パスワード**：メールのパスワード
   - **SMTPサーバー**：`v2008.coreserver.jp`（受信と同じ）
   - ポート／セキュリティは **ポート465／SSL（TLS）** にします。
   - 入力できたら「**次へ**」。
10. 同期頻度などの確認画面はそのまま「**次へ**」で進めます。
11. **名前**（差出人として表示される名前：例 ヴィレッジ）を入力し「**次へ**」で完了です。

> **うまくいかないとき（Android）**
> - 「サーバーに接続できません」と出る場合は、**ポート番号**と**サーバー名**の打ち間違いがないか確認してください。
>   （特にサーバー名は `coreserver.jp` で終わるものを使います。`village2024.jp` ではありません）
> - 993／465 で繋がらない場合は、受信 **143**／送信 **587**（暗号化は STARTTLS）に変えて再度お試しください。
> - 機種によっては「個人用（IMAP）」が「**IMAP**」とだけ表示される場合があります。同じ意味です。

## ④ 設定後の動作確認（送受信テスト）

設定が終わったら、必ず送受信の両方を確認してください。

1. **受信テスト**：別のメール（個人の Gmail など）から、設定したアドレス宛にメールを送り、
   スマホに届くか確認します。
2. **送信テスト**：スマホの設定したアドレスから、個人のメール宛に送り、届くか確認します。
   - 送信でエラーが出る場合は、**送信サーバー（SMTP）の認証**が原因のことが多いです。
     「ログインが必要／認証する」がオンで、ユーザー名・パスワードが入っているか確認してください。

## ⑤ よくある質問・注意点

| 症状・疑問 | 確認するポイント |
|-----------|----------------|
| パスワードが分からない | CoreServer 管理画面でメールアカウントのパスワードを再設定できます。 |
| 受信はできるが送信できない | 送信（SMTP）の「認証（ログイン）」がオフになっていないか確認。ポートを 465↔587 で切替。 |
| 送信はできるが受信できない | 受信（IMAP）のサーバー名・ポート（993）・暗号化（SSL/TLS）・パスワードを再確認。 |
| 「接続できない」と出る（ポート143） | `143` に `SSL/TLS` を選ぶと繋がりません。**993＋SSL/TLS**、または **143＋STARTTLS** に。 |
| 複数のスマホで同じメールを見たい | IMAP で設定すれば、複数端末で同じ受信トレイを共有できます（本マニュアルの方式）。 |
| 機種変更したらメールが消える？ | IMAP はサーバーにメールが残るため、新しい端末で同じ設定をすれば過去メールも見られます。 |
| サーバー名が分からない | 送受信とも `v2008.coreserver.jp`。CoreServer コントロールパネルでも確認できます。 |
